Layout Primatives

Custom tags inspired by Every Layout.

A collection of custom tags for layout elements to control the flow of your page. Based on Every Layout the goal is to accept that your website is not a poster and instead build primitives that make sure your sites content flows correctly regardless of device size.

Custom tags are used because div soup is bad. HTML allows you to name your own tags, so why not leverage it? often you'll see class="foo" instead. But what if Bar is a class as well? class="foo bar" may indeed be nonsensical. As such enforcing the layouts as distinct elements creates the nicest experience in my opinion.

I won't reiterate all the content of Every Layout. So go get Every Layout for real. Totally worth it.

Stack

Applies consistent spacing between sibling elements using the owl selector > * + * to make sure there isn't dead spacing at the end.

Properties --space

Basic Example

First item — no top margin Second item Third item Fourth item

      <layout-stack style="--space: var(--space-s);">
        <ui-card>First item — no top margin</ui-card>
        <ui-card>Second item</ui-card>
        <ui-card>Third item</ui-card>
        <ui-card>Fourth item</ui-card>
      </layout-stack>

Nested stacks with different spacing

Tight group A.1 Tight group A.2 Tight group B.1 Tight group B.2

Box

Adds padding and an optional border. All children inherit color.

Properties --padding --border-width --border-color

A boxed section

The box provides consistent internal spacing and a visible boundary. Useful for cards, callouts, and panels.

A warning box with thick border and custom background.


      <layout-stack style="--space: var(--space-s)">
        <layout-box style="--padding: var(--space-l); --border-color: var(--color-primary); --border-width: 2px;">
          <layout-stack style="--space: var(--space-s);">
            <h3>A boxed section</h3>
            <p>The box provides consistent internal spacing and a visible boundary. Useful for cards, callouts, and
              panels.</p>
          </layout-stack>
        </layout-box>

        <layout-box
          style="--padding: var(--space-m); --border-width: 5px; --border-color: var(--color-warning); background: var(--color-warning-subtle);">
          <p>A warning box with thick border and custom background.</p>
        </layout-box>
      </layout-stack>

Sidebar

Two-column layout that stacks on narrow containers. The first child is the sidebar, the second grows to fill.

Properties --sidebar-width --min-width --gutter

Sidebar

Navigation, filters, or supplementary content.

Main Content

This area grows to fill available space. When the container becomes too narrow, the sidebar stacks above.

Resize the browser window to see the intrinsic switching behavior — no media queries involved.


      <layout-sidebar style="--sidebar-width: 14rem; --gutter: var(--space-m);">
        <layout-box style="--border-color: var(--color-primary);">
          <layout-stack style="--space: var(--space-xs);">
            <h3>Sidebar</h3>
            <p>Navigation, filters, or supplementary content.</p>
          </layout-stack>
        </layout-box>
        <layout-box style="--border-color: var(--color-secondary);">
          <layout-stack style="--space: var(--space-xs);">
            <h3>Main Content</h3>
            <p>This area grows to fill available space. When the container becomes too narrow, the sidebar stacks above.
            </p>
            <p>Resize the browser window to see the intrinsic switching behavior — no media queries involved.</p>
          </layout-stack>
        </layout-box>
      </layout-sidebar>

Frame

Maintains a fixed aspect ratio and crops overflow content with object-fit: cover.

Properties --ratio

16 : 9

1 : 1

4 : 3

        <layout-stack style="--space: var(--space-2xs);">
