<ui-tabs> — Tab Navigation

Pure CSS tabs using radio inputs and labels. Three variants: default (bordered), pills, and underline. Implemented as a custom element for cleaner markup.

Structure

<ui-tabs>
  <input type="radio" name="demo" id="tab1" checked>
  <label for="tab1">Tab 1</label>
  <div data-content>Content 1</div>

  <input type="radio" name="demo" id="tab2">
  <label for="tab2">Tab 2</label>
  <div data-content>Content 2</div>
</ui-tabs>
Properties --tab-bg transparent --tab-bg-active var(--color-primary) --tab-color #666 --tab-color-active var(--color-canvas) --tab-border-color var(--color-border) --tab-border-width 2px --tab-padding 0.75rem 1.5rem --tab-radius 6px 6px 0 0

Default Tabs

Overview

This is the overview panel. Tabs use radio inputs under the hood, so only one can be active at a time.

The active tab has a border that connects to the content panel.

Features

  • Pure CSS — no JavaScript required
  • Keyboard accessible
  • Fully customizable with CSS custom properties
  • Three built-in variants

Code

The implementation uses the :checked pseudo-class and adjacent sibling selectors to show/hide content panels.

All radio inputs share the same name attribute to ensure mutual exclusivity.

Vanilla HTML Components

Native HTML elements that work without any JavaScript or custom CSS. These semantic elements provide built-in interactivity and accessibility.

Accordion with <details> and <summary>

The <details> element creates a disclosure widget that can be toggled open and closed. No JavaScript required.

<details>
  <summary>What is Semantastic?</summary>
  <p>A CSS toolkit with layout primitives and components.</p>
</details>

Basic Example

What is Semantastic?

Semantastic is a CSS toolkit featuring design tokens, composable layout primitives, CSS patterns, and pure CSS components. It emphasizes semantic HTML and CSS custom properties over classes and JavaScript.

How do I use the layout primitives?

Layout primitives are custom HTML elements like <layout-stack> and <layout-grid>. Just use them as tags in your HTML and configure them with CSS custom properties via inline styles or classes.

Do I need to register custom elements?

No! The layout primitives work with pure CSS using element selectors. There's no JavaScript custom element registration required.

Can details be open by default?

Yes! Add the open attribute to have a details element expanded by default, like this one.

Why use semantic HTML?

Semantic HTML improves accessibility, SEO, and code maintainability. Native elements like <details> come with built-in keyboard navigation and screen reader support.

Benefits of native components

  • Built-in accessibility
  • Better performance
  • Future-proof
  • Progressive enhancement
