Colors

we want to be semantic, doesn't me we can't look good.

Modern CSS has delightful tools for working with color. colors.css leverages OKLCH and color-mix to make good and consistent colors that only need a small handful swapped out to theme the site. Dark mode included.

colors.css has many colors defined. But less than a dozen core colors need to be defined to totally customize the looks.

Why OKLCH?

Traditional HSL is not perceptually uniform. Colors with the same lightness value can appear dramatically different in brightness:

HSL Blue hsl(240, 100%, 50%)
HSL Yellow
hsl(60, 100%, 50%) much brighter!

OKLCH fixes this. When you set lightness to 50%, all hues appear equally bright:

OKLCH Blue oklch(85% 0.2 240)
OKLCH Yellow
oklch(85% 0.2 100) Better!

Color Tokens

Pre-defined color tokens for common UI needs. Change these basics and everything will update via color-mix.

Primary
--color-primary
oklch(44% 0.095 250)
Secondary
--color-secondary
oklch(75% 0.08 220)
Success
--color-success
oklch(65% 0.23 142)
Warning
--color-warning
oklch(75% 0.17 90)
Error
--color-error
oklch(51% 0.2 31)
